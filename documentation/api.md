# API Documentation (OpenAPI/Swagger Specification)
## FeedLink AI: Production-Grade Intelligent Social Impact Platform

---

### 1. Global Authentication Security
- **Header Format**: `Authorization: Bearer <JWT_TOKEN>`
- **Endpoint Rules**: Except for specific public authentication paths (`/api/auth/**`), all routes require a valid Bearer token. Roles are verified at the gateway level.

---

### 2. Authentication APIs

#### 2.1 Register Hotel & Hostel
- **Endpoint**: `POST /api/auth/register/hotel`
- **Access**: Public
- **Request Body**:
  ```json
  {
    "email": "hotel@grandhyderabad.com",
    "password": "HotelPassword@123",
    "name": "Grand Hyderabad Hotel",
    "phone": "+919876543210",
    "address": "Gachibowli, Hyderabad",
    "latitude": 17.4483,
    "longitude": 78.3741
  }
  ```
- **Response** (200 OK):
  ```json
  {
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJob3R...",
    "refreshToken": "4a7b9c2d-e8f0-1234-5678-...",
    "email": "hotel@grandhyderabad.com",
    "role": "HOTEL",
    "name": "Grand Hyderabad Hotel"
  }
  ```

#### 2.2 Register NGO Partner
- **Endpoint**: `POST /api/auth/register/ngo`
- **Access**: Public
- **Request Body**:
  ```json
  {
    "email": "ngo@hungersaves.org",
    "password": "NgoPassword@123",
    "name": "Hunger Saves NGO",
    "phone": "+918765432109",
    "address": "Madhapur, Hyderabad",
    "latitude": 17.4411,
    "longitude": 78.3826,
    "ngoName": "Hunger Saves Foundation",
    "registrationNumber": "NGO-REG-98273-HYD",
    "serviceArea": "Cyberabad District"
  }
  ```
- **Response** (200 OK): Contains authentication JWT payload (Role: `NGO`), account activation remains `PENDING` until Admin approval.

#### 2.3 User Login
- **Endpoint**: `POST /api/auth/login`
- **Access**: Public
- **Request Body**:
  ```json
  {
    "email": "admin@feedlink.ai",
    "password": "Admin@123"
  }
  ```
- **Response** (200 OK): Returns JWT token, refresh token, role, email, and username.

#### 2.4 Token Refresh
- **Endpoint**: `POST /api/auth/refresh`
- **Request Header**: `Cookie: refreshToken=<REFRESH_TOKEN>` or JSON body `{ "refreshToken": "<TOKEN>" }`
- **Response** (200 OK):
  ```json
  {
    "token": "new-jwt-token...",
    "refreshToken": "new-rotated-refresh-token..."
  }
  ```

### 3. Food Donation APIs

#### 3.1 Create surplus listing
- **Endpoint**: `POST /api/donations`
- **Access**: `HOTEL`
- **Request Body**:
  ```json
  {
    "foodType": "Chicken Biryani",
    "category": "Prepared Meal",
    "quantity": 45,
    "description": "Premium surplus food from lunch buffet. Stored under refrigeration.",
    "pickupAddress": "Gachibowli Banquet Hall",
    "expiryTime": "2026-06-27T23:30:00",
    "latitude": 17.4483,
    "longitude": 78.3741,
    "freshnessScore": 92.5,
    "airecommendation": "Safe to consume. Distribute within 4 hours."
  }
  ```
- **Response** (201 Created): Returns persisted Donation JSON.

#### 3.2 List nearby available donations
- **Endpoint**: `GET /api/donations`
- **Access**: `NGO`, `ADMIN`, `VOLUNTEER`
- **Response** (200 OK): Array of active donations (Status: `AVAILABLE`).

#### 3.3 Claim donation
- **Endpoint**: `PUT /api/donations/claim/{id}`
- **Access**: `NGO`
- **Response** (200 OK): Updates status to `ACCEPTED` and returns the verification token code.

#### 3.4 Scan Handover Verification
- **Endpoint**: `POST /api/donations/verify`
- **Access**: `NGO` (scans code generated by Hotel)
- **Request Body**:
  ```json
  {
    "donationId": 12,
    "token": "verif-token-123-uuid"
  }
  ```
- **Response** (200 OK):
  ```json
  {
    "status": "success",
    "message": "Handover verified. Status updated to DELIVERED."
  }
  ```

### 5. Admin Console Management APIs

#### 5.1 Approve NGO registration
- **Endpoint**: `PUT /api/admin/approve-ngo/{id}`
- **Access**: `ADMIN`
- **Response** (200 OK): NGO entity with updated activation status `APPROVED` and linked user status `ACTIVE`.

#### 5.2 Suspend User Account
- **Endpoint**: `PUT /api/admin/suspend-user/{id}`
- **Access**: `ADMIN`
- **Response** (200 OK): Suspends login privileges.

---

### 6. Profile Management APIs

#### 6.1 View Profile Info
- **Endpoint**: `GET /api/profile`
- **Access**: Authorized Users
- **Response** (200 OK): Renders account detail parameters (address, phone coordinates, registration keys).
